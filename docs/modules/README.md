# 模块独立 Spec（第二层）

本目录放**单个业务模块**的高层说明：**边界、交互逻辑、验收标准**。不写实现细节、不粘贴大段源码（实现以代码与第一层内联注释为准）。

## 何时新增模块 Spec

- 某栏目从「占位页」演进为独立产品单元（如博客列表 + 详情 + 数据源）。
- 多人协作或 AI 长周期开发同一模块，需要固定验收口径。

## 文件命名

```text
module-<领域>-spec.md    # 例：module-blogs-spec.md、module-project-spec.md
```

## 推荐结构（复制后删节）

```markdown
# <模块名> Spec

## 1. 范围与边界
- 做什么 / 不做什么（与 docs/01 冲突时先改 01）

## 2. 用户可见行为与交互
- 页面、路由、关键操作、空态与错误态（高层描述）

## 3. 验收标准（可勾选）
- [ ] …
- [ ] …

## 4. 依赖与数据（概念层）
- 依赖哪些模块、数据从哪类来源来（不写具体函数名除非必要）

## 5. 与第一层的关系
- 业务常量、类型、关键配置预计落在哪些路径（一句话，不贴代码）
```
